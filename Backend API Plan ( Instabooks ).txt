Backend Data Schema ( Instabooks )

user
- name
- email
- addresses: [string]
- photo
- cart []
    - book id
    - order quantity
- id


books
- title
- description
- pages
- author
- price
- cover photo
- quantity
- genre
- id


orders
- user_id
- products
    - book id
    - order quantity
    - format
- id
- status ( shipped | pending | cancelled )
- price at purchase
- created at
- shipping address
- payment status

Backend API Plan ( Instabooks )


Auth 

GET /auth/google  ( redirect to google auth page )

Request body: none
Request Header: none
Response: none ( redirects to google auth page )
Response Header: none


GET /auth/google/callback  ( callback endpoint for google auth )

Request body: none
Request Header: none
Response Header: none
Response: 
- 200: Redirect to frontend with auth id in query parameters (e.g. 
    /auth/success?id=<auth id>)


GET /auth/google/verify  ( verify google auth callback for frontend )

Request query parameters:
- authId: string ( auth id from google auth callback )
Request body: none
Request Header: none
Response Header: none
Response: 
- 200: {
    "status":"success",
    "data":{
        "name": < user's name: "John Doe" >,
        "email": < user's email: "john.doe@example.com" >,
        "photo_url": < user's photo url: "https://example.com/photo.jpg" >,
        "addresses":[] ( array of user's saved addresses ),
        "cart":[],
        "access_token": < JWT token: "erdrd.yyfytfyu.uyfytfy" >,
        "expires_in": < JWT token expiration time in seconds: 3600 >,
    }
} ( refresh token is set in httpOnly cookie )
- 400: {
    "status":"error",
    error: {
        "message": < error message >,
        "code": < error code >
    }
}
- 404: {
    "status":"error",
    error: {
        "message": < error message >,
        "code": < error code >
    }
}


GET /auth/logout  ( logout endpoint )

Request body: none
Request Header: 
- Authorization: Bearer <JWT token>
Response Header: none
Response: 
- 200: {
    "status": "success",
    "data":{
        "message": "user logged out successfully"
    }
}
- 401: {
    "status":"error",
    error: {
        "message": < error message >,
        "code": < error code >
    }
} or "Unauthorized" if token is invalid ( based on passport middleware )


GET /auth/me  ( get current user information )

Request body: none
Request Header
- Authorization: Bearer <JWT token>
Response Header: none
Response:
- 200: {
    "status":"success",
    "data": {
        "user": {
            "name": < user's name: "Joshua Ligron" >,
            "email": < user's email: "joshuanathan086@gmail.com" >,
            "photo_url": < user's photo URL: "https://lh3.googleusercontent.com/a/***" >,
            "addresses": < user's addresses: [] >,
            "cart": < user's cart: [] >
        }
    }
}
- 401: {
    "status":"error",
    error: {
        "message": < error message >,
        "code": < error code >
    }
} or "Unauthorized" if token is invalid ( based on passport middleware )


POST /auth/refresh  ( refresh JWT token )

Request body: none
Request Header:
- cookie: refresh_token=<refresh token from httpOnly cookie>
Response Header: none
Response:
- 200: {
    "status": "success",
    "data": { 
        "access_token": < new JWT token: "erdrd.yyfytfyu.uyfytfy" >,
        "expires_in": 3600 ( expiration time in seconds )
    }
}
- 401: {
    "status":"error",
    error: {
        "message": < error message >,
        "code": < error code >
    }
} or "Unauthorized" if refresh token is invalid ( based on passport middleware )
- 404: {
    "status":"error",
    error: {
        "message": < error message >,
        "code": < error code >
    }
}


POST /auth/update?deletePhoto={boolean}  ( update user information )

Request body:
- FormData object containing updated user information (e.g. name, email)
Request Header:
- Authorization: Bearer <JWT token>
Response Header: none
Response:
- 200: {
    "status": "success",
    "data":{
        "user": {
            "name": < user's name: "Joshua Nathan Ligron" >,
            "email": < user's email: "joshuanathan086@gmail.com" >,
            "photo_url": < user's photo URL: "http://example.com/photo.jpg" >,
            "addresses": < user's addresses: [ "123 Main St, Anytown, USA", ... ] >,
            "cart": < user's cart: [] >
        }
    }
}
- 400: {
    "status":"error",
    error: {
        "message": < error message >,
        "code": < error code >
    }
}
- 401: {
    "status":"error",
    error: {
        "message": < error message >,
        "code": < error code >
    }
} or "Unauthorized" if token is invalid ( based on passport middleware )


Users

POST /user/cart             ( add book to cart )

Request body:
- JSON object containing book id and order quantity
Request Header
- Authorization: Bearer <JWT token>
Response Header: none
Response:
- JSON object containing updated cart information

PUT /user/address          ( add new address )

Request body:
- JSON object containing new address information
Request Header
- Authorization: Bearer <JWT token>
Response Header: none
Response:
- JSON object containing updated addresses information


Books

GET /books?limit={number}  ( get all books )

Request body: none
Request Header: none
Response Header: none
Response:
- JSON array of book objects


GET /books/{id}  ( get book by id )

Request body: none
Request Header: none
Response Header: none
Response:
- JSON object containing book information


POST /books  ( create a new book ) [admin only]

Request body:
- FormData object containing book information (e.g. title, author, price, cover photo file)
Request Header: 
- Authorization: Bearer < admin key >
Response Header: none
Response:
- JSON object containing created book information


PUT /books/{id}  ( update book information ) [admin only]

Request body:
- FormData object containing updated book information (e.g. title, author, price, cover photo file)
Request Header:
- Authorization: Bearer < admin key >
Response Header: none
Response:
- JSON object containing updated book information


DELETE /books/{id}  ( delete a book ) [admin only]

Request body: none
Request Header:
- Authorization: Bearer < admin key >
Response Header: none
Response:
- JSON object confirming deletion with message


Orders

POST /orders/checkout  ( create a new order )

Request body:
- JSON object containing order information (e.g. products, shipping address)
Request Header: 
- Authorization: Bearer <JWT token>
Response Header: none
Response:
- Redirect to payment gateway with order details


GET /orders/confirm?reference={order_id}  ( confirm order after successful redirect from payment gateway )

Request body: none
Request Header: none
Response Header: none
Response:
- Redirect to order confirmation frontend page with order details


GET /orders?limit={number}  ( get all orders for current user )

Request body: none
Request Header:
- Authorization: Bearer <JWT token>
Response Header: none
Response:
- JSON array of order objects for the current user


GET /orders/{id}  ( get order by id for current user )

Request body: none
Request Header
- Authorization: Bearer <JWT token>
Response Header: none
Response:
- JSON object containing order information if the order belongs to the current user


GET /orders/admin?limit={number}  ( get all orders for admin )[admin only]

Request body: none
Request Header:
- Authorization: Bearer < admin key >
Response Header: none
Response:
- JSON array of all order objects





Footnote:
- logging and observability design should include user_id on 
  protected endpoints for better traceability, request_id on all endpoints for better correlation, 
  , error logging with stack traces for easier debugging, order_id on order related endpoints 
  for better traceability of order processing and summarized logs to prevent log flooding 
  while still capturing important events and errors.
